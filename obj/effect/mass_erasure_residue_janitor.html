Vars

consecutive_empty_jumpsHow many consecutive frontier jumps this janitor has made without finding any residue to propagate through.
janitor_activeWhether this janitor should keep processing.
local_residue_turfsLocal void turfs containing janitor-eligible residue in the current cleanup pocket.
max_attempts_per_tickMaximum number of local residue frontier entries this janitor will evaluate per processing tick.
max_empty_jumpsHow many consecutive frontier jumps this janitor may make without finding any residue before it self-deletes.
owning_controllerController coordinating this janitor's lifecycle and relocation.
residue_cleanup_typesNarrow allowlist of residue objects this janitor is allowed to clean from void tiles.
turfs_erased_per_tickHow many residue-bearing void turfs this janitor may clean per processing tick. This is assigned by the controller on spawn so VV can tune it centrally there.

Procs

DestroyUnregisters this janitor from the controller and clears its local residue cache.
InitializeFinishes janitor initialization without seeding, because the owning controller is attached immediately after creation.

find_visible_residue_sourceReturns the first residue-bearing void turf visible from this janitor's current frontier position.
has_viable_frontierReturns TRUE when this janitor still has a residue frontier queued in its current cleanup pocket.
move_to_frontier_sourceMoves the janitor to a fresh active frontier turf and seeds a local cleanup pocket from the first visible residue match.
process_cleanup_tickCleans local residue-bearing void turfs, or relocates this janitor when its current pocket runs dry.
prune_local_residue_frontierRemoves stale or self-referential residue entries so the janitor does not orbit invalid cleanup work.
seed_local_residue_frontierSeeds the janitor's local frontier from the first visible residue-bearing void turf in its new frontier pocket.
validate_host_controllerRemoves manually spawned janitors that never received a valid host controller.
